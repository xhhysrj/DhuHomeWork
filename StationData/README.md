本文档详细说明了上海地铁路径规划系统所使用的核心数据文件：Station.csv 和 Edge.csv。这两个文件共同构建了整个地铁网络的拓扑结构，是实现路径规划算法的基础。

---

### 1. Station.csv - 站点信息文件

该文件定义了上海地铁网络中的所有站点及其属性。系统将根据此文件加载站点数据，并依据 `status` 字段判断站点是否可用。

#### 1.1 文件结构

| 字段名           | 中文说明     | 描述 |
|------------------|--------------|------|
| station_id       | 站点ID       | 站点的唯一数字标识符。注意：换乘站在不同线路上拥有不同的ID，例如，人民广场站作为1、2、8号线的换乘站，会有三个独立的条目和ID。 |
| station_name     | 站点名       | 站点的官方中文名称。 |
| line             | 所属线路     | 站点所属的地铁线路，例如“1号线”。 |
| status           | 运营状态     | 表示站点的当前状态，值为 `"active"` 或 `"inactive"`。路径规划算法会跳过所有 `"inactive"` 状态的站点。初始状态下，所有站点均为 `"active"`。 |

#### 1.2 数据范围

- 线路覆盖：包含上海地铁1号线至18号线、浦江线及市域机场线的所有站点。
- 站点总数：包含所有换乘站在内，共计约 525 个站点条目。

---

### 2. Edge.csv - 线路连接文件

该文件描述了地铁站点之间的连接关系（边）、通行耗时以及所属线路。它是构建图邻接表的数据来源。

#### 2.1 文件结构

| 字段名            | 中文说明     | 描述 |
|-------------------|--------------|------|
| start_station_id  | 起点站ID     | 边的起始站点ID，与 Station.csv 中的 station_id 相对应。 |
| end_station_id    | 终点站ID     | 边的结束站点ID，与 Station.csv 中的 station_id 相对应。 |
| line              | 所属线路     | 该段线路所属的地铁线路名称（如“1号线”）。对于换乘，此字段值为“transfer”。 |
| direction         | 运行方向     | 仅用于4号线，用于区分“inner”和“outer”。其他线路留空。 |
| time              | 通行时间     | 从起点站行驶至终点站的耗时，单位为分钟。支持浮点数（如 2.5）。 |

#### 2.2 关键规则

- 有向边处理：由于A站到B站与B站到A站的行驶时间可能不同，所有站点间的连接都按有向边处理，即每一段物理线路都对应文件中的两条记录。
- 换乘处理：不同线路间的换乘被视为一条特殊的边，其 Line 字段为 `"transfer"`，Time 字段统一设置为 `5` 分钟。
- 边的总数：包括所有方向的线路和换乘在内，总共约 1226 条边。

---

### 3. 文件格式与编码

- 两个文件均为 CSV (逗号分隔值) 格式。
- 推荐使用 UTF-8 编码保存，并在文件开头添加 BOM 标识以确保中文字符正确读取。
